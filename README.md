# Lead Finder Brasil — Landing Pages e Demonstrações

Catálogo estático, responsivo e comercial da **Lead Finder Brasil**, criado para apresentar serviços de desenvolvimento de landing pages para empresas, profissionais e pequenos negócios de **qualquer segmento**, com atendimento online em todo o Brasil.

**Site público:** https://brunoferreirasalustiano.github.io/lead-finder-demos/

> A versão documentada neste README está sendo preparada na PR #12. O conteúdo público pode permanecer na versão anterior até o merge na `main` e a conclusão do deploy do GitHub Pages.

## Posicionamento

A Lead Finder Brasil oferece landing pages sob medida para negócios de todos os segmentos.

As demonstrações atuais usam quatro categorias apenas como exemplos visuais:

- barbearias;
- oficinas mecânicas;
- restaurantes;
- prestadores de serviços.

Esses exemplos **não limitam o público atendido**. A estrutura pode ser adaptada para comércio, saúde, beleza, alimentação, manutenção, educação, consultoria, profissionais autônomos, serviços locais e outros modelos de negócio.

Mensagem central:

> Landing pages profissionais para transformar visitas em conversas no WhatsApp.

## WhatsApp comercial

Todos os links comerciais usam diretamente o endereço autorizado:

```text
https://wa.me/5519971519337
```

Cada página possui uma mensagem específica em `data-message` para identificar a origem do interesse.

Exemplos:

- home: interesse no Pacote Essencial;
- barbearia: interesse em uma página para barbearia;
- oficina: interesse em uma página para oficina;
- restaurante: interesse em uma página para restaurante;
- prestador: interesse em divulgação de serviços.

### Resiliência

Os links `wa.me` estão presentes diretamente no HTML. Assim, os botões continuam funcionando mesmo se o JavaScript não carregar.

O JavaScript compartilhado:

- lê `data-message`;
- gera a URL codificada;
- usa apenas o número autorizado;
- adiciona `target="_blank"`;
- adiciona `rel="noopener noreferrer"`;
- melhora o rótulo acessível;
- não envia mensagens automaticamente.

O visitante sempre decide se deseja enviar a mensagem no WhatsApp.

O site não usa:

- WhatsApp Cloud API;
- automação de WhatsApp Web;
- disparo em massa;
- webhook;
- provider de envio;
- confirmação automática de mensagem.

## Publicação

A workflow `.github/workflows/pages.yml` executa:

1. checkout sem persistência de credenciais;
2. validação estática;
3. preparação de artefato por allowlist;
4. upload do artefato;
5. publicação no GitHub Pages após alterações aprovadas na `main`.

Em pull requests, somente a validação é executada. O deploy é ignorado corretamente até o merge.

O script `scripts/prepare-pages-artifact.mjs` publica somente:

- `index.html`;
- `.nojekyll`;
- `robots.txt`;
- `sitemap.xml`;
- `assets`;
- `barbearia`;
- `oficina`;
- `restaurante`;
- `prestador-servicos`.

Arquivos internos como `.git`, `.github`, `README.md` e `scripts` não são incluídos no artefato público.
